# Skill: git-commit

ステージング済みの差分を確認し、コミットを実行する。
コミットが失敗した場合（GPG 署名エラーなど）に限り、`git commit -m "..."` の形式でコマンドを返す。

## トリガー

- `/git-commit` を実行したとき

## 絶対に守るルール

- **`git add` は絶対に実行しない。いかなる理由があっても禁止。**
- **pre-commit フックや GPG 署名をスキップするフラグは絶対に使用しない。** 具体的には `--no-verify` / `-n`、`--no-gpg-sign`、`-c commit.gpgsign=false` など。署名エラーやフック失敗は後述の手順で根本原因に対処する。
- ステージング済みの差分が空の場合は、即座に「ステージングされた変更がありません」とだけ伝えて処理を終了する。それ以外の行動は一切取らない。

## 実行手順

1. `git diff --cached --stat` を Bash ツールで実行し、出力が空かどうかを確認する
2. 出力が空の場合は「ステージングされた変更がありません」とだけ伝えて**即終了する**（add の提案・確認・その他の処理も一切しない）
3. `git diff --cached` を Bash ツールで実行し、差分の全内容を取得する
4. 差分の内容からコミットメッセージを作成する
5. `git commit -m "..."` を Bash ツールで実行してコミットを試みる
6. コミットが成功した場合はその旨を伝えて終了する
7. コミットが失敗した場合:
   - GPG 署名エラーの場合は、`~/.gnupg/gpg-agent.conf` の存在を確認する
     - ファイルがない、またはキャッシュ設定がない場合は以下のコマンドも合わせて返す:
       ```bash
       cat >> ~/.gnupg/gpg-agent.conf << 'EOF'
       default-cache-ttl 3600   # 最後の操作から1時間（3600秒）キャッシュ
       max-cache-ttl 86400      # キャッシュの最大保持時間は24時間（86400秒）
       EOF
       gpgconf --kill gpg-agent
       ```
       また、macOS の場合は `pinentry-mac`（`brew install pinentry-mac`）の利用も勧める
   - 以下の形式でそのまま実行できるコミットコマンドを返す:

```
git commit -m "件名"
```

複数の変更種別をまとめる場合は body を追加してもよい:

```
git commit -m "件名

- 変更1の説明
- 変更2の説明"
```

## コミットメッセージのルール

- 件名は英語・命令形（"Fix", "Add", "Update", "Remove", "Refactor" など）で書く
- 件名は 72 文字以内
- body は「なぜ変えたか」ではなく「何を変えたか」を箇条書きで記載する（差分から自明な場合は省略可）

## 出力形式

- 成功時: 1行でコミット完了を伝える
- 失敗時: コマンドのみをコードブロックで返す。説明文は最小限にする
